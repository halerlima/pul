# Contributing to PUL

Thank you for your interest in contributing to the **Protocol for Universal Settlement (PUL)**.

PUL is an open community project. Our goal is to build an open protocol for universal value transfer, similar to how TCP/IP became the foundation of the Internet.

We welcome contributions from developers, researchers, cryptographers, security experts, economists, architects, students, and anyone interested in helping build an open standard.

---

# Before You Start

Please read:

* README.md
* PUL-0001
* Existing discussions and issues

Understanding the project's vision before proposing changes helps keep the protocol consistent.

---

# Ways to Contribute

You can contribute by:

* Improving documentation
* Reporting issues
* Suggesting protocol improvements
* Reviewing specifications
* Writing reference implementations
* Developing SDKs
* Creating examples
* Improving security
* Writing tests
* Participating in technical discussions

Every contribution matters.

---

# Protocol Proposals

Major changes to the protocol should begin with a discussion.

Please open an Issue describing:

* The problem
* The proposed solution
* Why it benefits the protocol
* Possible drawbacks

Large protocol changes may later become a new PUL specification.

---

# Pull Requests

Before submitting a Pull Request:

* Keep changes focused.
* Follow the existing documentation style.
* Explain the motivation.
* Keep commits clean and descriptive.

---

# Coding Standards

Reference implementations should prioritize:

* Simplicity
* Readability
* Security
* Performance
* Portability

---

# Documentation

Documentation is considered as important as code.

Whenever possible:

* Explain the reasoning.
* Provide examples.
* Keep language clear and concise.

---

# Community Principles

We expect contributors to:

* Be respectful.
* Be constructive.
* Focus on technical discussions.
* Welcome different ideas.
* Help improve the protocol.

---

# Governance

PUL is designed to be community-driven.

No individual, company, or institution should have exclusive control over the protocol.

The protocol evolves through open discussion and community consensus.

---

# Thank You

By contributing to PUL, you are helping build an open standard for universal value transfer.

Thank you for being part of the project.
